Xray Markdown Evidence Builder

Built by SHIKDER SHONDHI

Reusable project utility for loading Markdown or CSV test cases, pasting screenshots against each TC, saving evidence locally, and exporting/importing evidence bundles for later Xray use.

User Manual

Fresh Setup

Node.js is required. If node -v or npm -v does not work, install Node.js first. For a fresh copy, run setup once, check the setup, then start the app.

  1. npm run setup
  2. npm run doctor
  3. npm run evidence:workflow

The button works when this page is opened from the local server. If this page was opened directly as a file, run the commands above from the repo terminal.

Daily Workflow

  1. Run npm run evidence:workflow from the repo.
  2. Load a Markdown or CSV testcase file.
  3. Paste screenshots into the correct TC paste area.
  4. Add notes and Actual Result text when needed.
  5. Reorder screenshots with Up or Down.
  6. Use Copy TC Evidence for one reliable combined image.
  7. Use copy seperatly to try separate screenshot clipboard items, then paste once into Jira Xray.

Clipboard to Jira Xray

  • Copy copies one screenshot, including its note when filled.
  • Copy TC Evidence creates one image with Actual Result first, then notes and screenshots in visible order.
  • copy seperatly writes screenshots as separate clipboard image items when the browser and OS support it.
  • If Jira Xray pastes only one item or the clipboard is rejected, use Copy TC Evidence.

Jira Xray Workflow

  1. Run npm run evidence:workflow from the repo to start the local server and open this page.
  2. Open Settings and save the exact test execution summary under Xray Workflow.
  3. Click Start Workflow in one TC card, or start all TCs with evidence from the workbench header.
  4. Complete Atlassian login and 2FA in the Playwright browser within 3 minutes if prompted.
  5. The workflow matches Xray by testcase title/name. The local TC-001 label is not sent to Xray.

GitHub Gist Use

  1. Create a Personal Access Token with gist scope.
  2. Open Settings.
  3. Enter GitHub username and token.
  4. Save Settings to create or reuse the private Gist ID.
  5. Export Settings to keep a portable Gist sync setup file. Include token only when you need a full restore file.
  6. Import Settings fills the Gist fields; click Save Settings to apply them in this browser.
  7. Use Push to Gist to upload this browser's workspaces.
  8. Use Sync from Gist on another browser to pull them back.
  9. Gist sync stores the evidence JSON, including screenshot data saved in the workspace.

Backup

Browser storage can be cleared by browser cleanup or profile changes. Use Export Evidence at the end of important sessions and Import Evidence when you need to restore that JSON backup.

Filtering Test Cases

Use the evidence filter beside search to show all test cases, only test cases with saved screenshots, or only test cases still missing screenshots.
