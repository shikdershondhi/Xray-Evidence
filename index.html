Built for repeatable evidence work.

The app keeps testcase content, screenshots, local workflow state, and setup checks close together so evidence runs are easy to repeat from a fresh copy.

MD

Markdown and CSV import

Bring testcase steps and expected results from Markdown or CSV files, then preserve the source structure while you attach evidence.

CP

Clipboard evidence

Copy a merged testcase evidence image or separate screenshot items with notes when browser clipboard support allows it.

PW

Local Playwright workflow

Start the local Node service and let Playwright drive the browser flow that pastes evidence into Jira Xray.

LO

Local-only data

Test evidence, screenshots, backups, Jira session state, and workflow payloads stay in your browser and local project folder.

GS

Gist sync

Optional Gist-backed workspace sync helps move evidence state between machines without introducing a custom hosted service.

CL

Uploaded evidence cleanup

After evidence is uploaded to Xray, clear saved screenshot files for uploaded TCs while keeping testcase status and workspace metadata.

FS

Fresh-copy setup

Setup and doctor commands check Node.js, dependencies, Playwright Chromium, launch scripts, and workflow readiness.
